# cccat 개인정보 안내

cccat은 100% 로컬에서 동작합니다. 네트워크 요청은 0회입니다 — 어떤 서버로도 데이터를
전송하지 않습니다. 이 문서는 실제로 저장되는 것과 저장되지 않는 것을 코드 기준으로
설명합니다.

## 저장하는 것

hook 이벤트가 발생할 때마다 `lib/classify.js`가 페이로드에서 다음만 추출합니다.

- **도구 종류**: `Read`, `Bash`, `Edit` 등 어떤 도구가 호출됐는지
- **파일 확장자**: `.js`, `.py`, `.md` 등 (파일 전체 경로가 아닌 확장자만)
- **명령 카테고리**: `git`, `testing`, `building`, `db` 등 Bash 명령의 앞부분 패턴을 미리
  정의된 규칙과 대조해 분류한 카테고리 (명령어 원문이 아님)
- **키워드 매칭 태그**: 프롬프트나 요약 텍스트를 정규식으로 매칭해 얻은 태그
  (`debug`, `test`, `refactor` 등) — 매칭에만 사용하고 원문은 즉시 버립니다
- **학습 기록**: 어떤 표현을 몇 번 봤는지, 언제 다음 복습이 예정됐는지, 저장했는지 여부

이 모든 데이터는 `~/.cccat/` 아래 JSON 파일로만 저장됩니다.

## 저장하지 않는 것

- 프롬프트 원문
- 파일 내용
- 파일 전체 경로 (확장자만 추출하고 버립니다)
- 환경변수
- 비밀키, 토큰, 인증정보
- Claude Code와의 대화 내용 전체

`lib/classify.js`의 `tagsFromText()`는 텍스트를 최대 2000자까지만 받아 정규식으로 태그를
뽑아낸 뒤 그 텍스트 자체는 어디에도 저장하지 않습니다. Bash 명령도 앞 500자만 패턴
매칭에 사용하고 저장하지 않습니다.

## 저장 위치

```
~/.cccat/
  config.json     설정 값
  state.json      현재 활동 상태, 최근 태그, 오늘 통계, 일시정지 여부
  history.json    표현별 학습 기록 (본 횟수, 다음 복습일, 저장 여부)
  backup/         설치 시 백업된 ~/.claude/settings.json
  packs/          사용자가 추가한 콘텐츠 팩 (선택)
```

터미널에서 직접 확인할 수 있습니다: `cccat config list`, `cccat stats`, `cat ~/.cccat/state.json`.

## 삭제

```sh
cccat reset            # 학습 기록(history.json)만 초기화
cccat reset --all       # 학습 기록 + 상태(state.json) 초기화
cccat uninstall --purge # 제거 + ~/.cccat 전체 삭제 (config, 백업, 사용자 팩 포함)
```

또는 직접: `rm -rf ~/.cccat`

## 왜 이렇게 설계했나

cccat의 핵심 기능(작업 맥락에 맞는 표현 추천)은 "지금 파일을 편집 중인지, 테스트를 돌리는
중인지, git 작업 중인지" 같은 큰 범주만 알면 충분합니다. 파일 내용이나 프롬프트 원문까지
볼 이유가 없기 때문에, 애초에 `lib/classify.js` 단계에서 걸러내고 그 이후 어떤 모듈에도
원문 데이터가 전달되지 않도록 만들었습니다.
